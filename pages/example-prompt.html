💬Prompt · Example

What a well-structured one-off prompt looks like.

Prompt

What it is

A specific request you write fresh each time and send into the chat. You're fully in control of the wording, the scope, and the output format. Use one when the task varies, you only need it once, or you want full editorial control.

Where it lives

In the chat window. Typed and sent — no file, no slash command.

Example

A focused PR review prompt:

Review the changes in this PR for security issues only.

Focus on:
- Authentication and session handling
- SQL injection or unsafe query construction
- Secrets accidentally committed
- Input validation gaps on API routes

Skip:
- Code style or formatting
- Naming conventions
- General refactoring suggestions

Output format: a markdown list grouped by file. Each finding includes
the file path, line number, severity (high / medium / low), and a
one-sentence explanation.

If you find nothing concerning, say so explicitly — don't invent issues.
What makes this good
  • Single, narrow goal. "Security issues only" — not "review this PR" (which would dump style nits, naming opinions, and refactor ideas).
  • Explicit "focus on" list. The AI knows exactly what to look for.
  • Explicit "skip" list. Prevents scope creep into things you don't care about right now.
  • Defined output format. Consistent structure makes the response scannable.
  • Exit clause. "If nothing, say so" stops the AI from inventing problems to seem useful.
  • No politeness padding. Direct and structured — the AI doesn't need pleasantries.
Watch out for
  • Vague prompts give vague results. "Make this better" or "review this code" leaves the AI guessing what you actually care about.
  • "Don't ___" is weaker than "do X instead". Negative instructions are easier for the model to drift from than positive ones.
  • One prompt, one task. Multi-task prompts ("review for security AND suggest refactors AND improve naming") usually produce shallow results on all of them.
  • Don't paste a wall of code without context. Tell the model what the code is for and what you're trying to achieve before pasting.
  • Specify output format if you care about it. Otherwise you'll get prose when you wanted a list, or a list when you wanted a table.
