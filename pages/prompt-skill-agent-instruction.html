Prompt · Skill · Agent · Instruction

When you face a problem — which path do you take?

Agents Prompting Learned

The one-sentence version

There are four ways to work with AI — and the right one depends on how often the task repeats and how much autonomy the AI needs to complete it.

The four paths

📋
Instruction
CLAUDE.md · system prompt

Persistent context that applies silently in every session — without you thinking about it. Rules, conventions, preferences that should always be true.

I keep having to remind the AI about the same thing…
Every project has its own conventions and the AI keeps missing them.
There's a fact about my setup or preferences that should never be forgotten.
💬
Prompt
one-off · you write it fresh

A specific request you write each time. You're fully in control. The task varies, the context changes, and you only need it once.

I need to ask the AI something specific right now…
This task is unique to today — I won't need it again.
I'm exploring an idea and want full control over the wording.
⚡
Skill
/slash command · repeatable shortcut

A prompt you've written once, triggered with a slash command. You still initiate it manually — but the instructions are pre-written and consistent every time.

I keep typing the same prompt over and over…
Every time I open a PR, I run the same checklist.
I want this to be one keystroke instead of a paragraph.
🤖
Agent
multi-step · tools · autonomous

You give it a goal. It figures out the how — searching, reading files, writing code, making decisions across multiple steps without you guiding each one.

I need it to figure out HOW to do this, not just what to say…
It should search the codebase, decide what to fix, then fix it.
I want to hand off a goal and walk away.

Key insight These aren't mutually exclusive. A well-crafted Instruction (CLAUDE.md) sets the stage, a Skill kicks off a task, and that task might spin up an Agent — all in the same session. Think of them as layers, not alternatives.

Watch out for
  • Don't reach for an Agent when a Prompt would do. Agents are powerful but slower and harder to predict. Start simple — escalate only when the task genuinely needs autonomy and tools.
  • Instructions are invisible. When the AI behaves unexpectedly, check CLAUDE.md or the system prompt first — it's probably quietly steering things in a way you forgot about.
  • Long Instructions cost tokens on every message. Keep CLAUDE.md focused on what actually matters; don't dump every preference you've ever had.
  • Skills can rot. If your workflow changes but the skill doesn't, you'll silently get stale results. Treat them like code — review and update them.
  • Agents amplify mistakes. A vague goal becomes a confidently wrong multi-step workflow. Be specific about success criteria before you let one loose.
  • Don't put fast-changing context into Instructions. They're for stable rules, not today's task state. Use Prompts for anything ephemeral.
