A repeatable prompt, saved as a slash command.

Skill

What it is

A prompt you've written once, saved to disk, and now trigger with a slash command like /review-pr. You still kick it off manually, but the instructions are pre-written and consistent every time.

Where it lives

  • Project: .claude/skills/<name>.md
  • Global: ~/.claude/skills/<name>.md

Example

A /review-pr skill — the file you'd put at .claude/skills/review-pr.md:

---
name: review-pr
description: Review the current branch's changes against main for security and correctness issues
---

Review the current branch's changes against main:

1. Run `git diff main...HEAD` to see what changed
2. For each modified file, identify:
   - Security concerns (auth, SQL, secrets, input validation)
   - Correctness bugs (off-by-one, null handling, race conditions)
   - Test coverage gaps for the new logic

Output:
- Group findings by file
- For each finding include: `file:line`, severity (high / medium / low),
  and a one-line explanation
- If nothing concerning, say "No issues found." — don't invent feedback

Skip: code style, formatting, naming conventions.
What makes this good
  • Frontmatter name and description. The description is what shows up in /help — make it specific so future-you remembers what it does.
  • Concrete first step. "Run git diff main...HEAD" gives the AI a clear starting action.
  • Defined output structure. Same shape every time, so you can scan results quickly.
  • Explicit "skip" list. Prevents scope creep into things you don't want.
  • Exit clause. "No issues found" prevents the AI from inventing feedback to seem useful.
  • Short. A skill that fits on one screen is one you'll actually maintain.
Watch out for
  • Skills can rot. If your workflow changes (new lint tool, new branch convention) but the skill doesn't, you'll silently get stale results. Treat them like code — review them.
  • The description triggers it. Vague descriptions ("Reviews things") mean you won't remember when to use it. Be specific.
  • Skills are saved prompts — same pitfalls apply. Vague instructions in the body produce vague output, every single time.
  • Don't proliferate. 50 skills you can't remember is worse than 5 you use weekly. Delete unused ones.
  • One skill, one job. Don't combine unrelated tasks into a mega-skill — split them.
